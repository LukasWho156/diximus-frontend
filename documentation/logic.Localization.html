Localization

logic. Localization

A class that takes care of localizing short strings. Currently supports English and German. Other translations can be easily added by creating corresponding JSON files.

localize(key, …replacementsopt) → {string}

Source:
  • logic/localization.js, line 90

Localize a message using a message key. The corresponding messages are saved in the language's JSON file. If a message cannot be found in the current language, default to the English message.

Some messages depend on input parameters. In this case, provide them as additional arguments after the key argument.

Parameters:
Name Type Attributes Description
key string

the message's key
replacements string <optional>
<repeatable>

within the messages, replace '{0}', '{1}' and so forth with these parameters.
Returns:

the localized message including replacements

string

localizeObject(object) → {string}

Source:
  • logic/localization.js, line 113

Use an object to do localization. The provided object should be structured such that the localizations are string properties of the object. Example:

cat = {
  en: 'cat',
  de: 'Katze',
}

This is mainly used to localize database object, as they cannot be caught in the translation file.

Parameters:
Name Type Description
object object

an object that contains the translations
Returns:

the localized version of the object.
